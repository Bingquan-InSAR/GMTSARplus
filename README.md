## рџљЂ run_sbas.sh

**End-to-end SBAS runner based on GMTSAR+.**

This script wraps the complete workflow into a single command, including:

1. AOI bbox extraction from a vector file (KML)  
2. Sentinel-1 burst/SAFE downloading via `download_s1.py` в†’ `burst2stack`  
3. precise orbit download via `eof`  
4. SBAS processing chain:  
   `pSAR_gmtsar_s1.py` в†’ `meta_creator.py` в†’ `gpkg_wrapper.py`

---

## рџ§ѕ Usage

```text
usage: run_sbas.sh [-h|--help]
                   --kml <aoi.kml)>
                   --st <YYYYMMDD> --ed <YYYYMMDD>
                   --rel_orbit <int>
                   --tmpbase <days>
                   --rlook <int> --azlook <int>
```

---

## рџ“ќ Description

Run a full SBAS workflow (download в†’ SBAS processing в†’ export) in one command.

- The AOI is read from `--kml`.
- The script computes the AOI bounding box in **EPSG:4326**, then formats it as:  
  `west,east,south,north` (required by `pSAR_gmtsar_s1.py -roi`).
- Downloaded Sentinel-1 data are stored under `./<outdir>` in the working directory.
- Precise orbit files are downloaded to:  
  `$S1_ORB/aux_poeorb`.

## рџ“¦ Outputs

Generated under `workdir`:

- `sbas/output.csv`  
  SBAS results produced by the processing chain

- `./output.csv`  
  Copied from `sbas/output.csv`

- `./output.gpkg` and `./output.xml`  
  Final export products generated by `gpkg_wrapper.py`
